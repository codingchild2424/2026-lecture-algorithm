## 도입 (3분)

> "지난 주에는 복잡도를 측정했습니다. 오늘은 정렬 알고리즘을 직접 구현하고 성능 차이를 눈으로 확인합니다. 선택 정렬, 버블 정렬, 삽입 정렬 -- 모두 O(n^2) -- 을 코딩한 뒤, 병합 정렬과 퀵 정렬 -- O(n log n) -- 을 구현합니다. 그다음 모든 알고리즘을 나란히 벤치마크하여 O(n^2)이 감당할 수 없는 지점을 확인합니다. 마지막으로 정렬 속도가 사용자 경험에 직접 영향을 주는 웹 애플리케이션에서 이를 체험합니다."

## Type A -- 알고리즘 구현

### A-1: 기본 정렬 구현 (10분)

#### 진행 방법

1. **간단 소개 (2분):**
   > "`examples/a1_basic_sorts.py`를 열어보세요. TODO가 표시된 세 함수가 있습니다: 선택 정렬, 버블 정렬, 삽입 정렬. 각 함수는 리스트를 받아 정렬된 새 리스트를 반환합니다. 모두 O(n^2) 알고리즘입니다 -- 구현은 쉽지만 큰 입력에서는 느립니다. 코드를 채우고 테스트를 실행하세요."

2. **학생 실습 (6분):**
   - 선택 정렬 구현: 최솟값을 찾아 앞으로 교환, 반복.
   - 버블 정렬 구현: 인접 원소를 반복적으로 비교하고 교환.
   - 삽입 정렬 구현: 각 원소를 정렬된 부분의 올바른 위치에 삽입.
   - `python examples/a1_basic_sorts.py`를 실행하여 정확성을 확인합니다.

3. **체크포인트 (2분):**
   > "세 알고리즘 모두 올바른 정렬 결과를 출력해야 합니다. 테스트가 통과하면, 세 알고리즘 모두 이중 루프를 가지고 있다는 점에 주목하세요 -- O(n^2)은 거기서 나옵니다. 이 셋 중에서는 거의 정렬된 데이터에서 삽입 정렬이 실제로 가장 빠릅니다. 기억해 두세요."

---

### A-2: 고급 정렬 구현 (15분)

#### 진행 방법

1. **간단 소개 (3분):**
   > "이제 `examples/a2_advanced_sorts.py`를 여세요. 여기서 병합 정렬과 퀵 정렬을 구현합니다. 둘 다 평균 O(n log n)이지만 동작 방식이 다릅니다. 병합 정렬은 배열을 반으로 나누고, 각 반을 재귀 정렬한 뒤 합칩니다. 퀵 정렬은 피벗을 골라 분할한 뒤 양쪽을 재귀 정렬합니다. TODO를 채워보세요."

2. **학생 실습 (10분):**
   - 병합 정렬 구현: 분할, 재귀, 정렬된 두 반쪽 합치기.
   - 퀵 정렬 구현: 피벗 선택, 분할, 양쪽 재귀.
   - 흔한 막히는 부분: 병합 정렬의 합치기 단계와 퀵 정렬의 분할 로직.
   - 작은 예시(예: [3, 1, 4, 1, 5])를 종이에 추적해 보도록 권장합니다.

3. **체크포인트 (2분):**
   > "병합 정렬과 퀵 정렬 모두 올바른 정렬 결과를 내야 합니다. 핵심 차이: 병합 정렬은 항상 균등하게 나누므로 O(n log n)이 보장됩니다. 퀵 정렬은 피벗 선택에 따라 최악이 O(n^2)이지만 평균은 O(n log n)입니다. 실무에서는 캐시 효율이 좋아서 퀵 정렬이 더 빠른 경우가 많습니다."

### A-3: 벤치마크 (10분)

#### 진행 방법

1. **간단 소개 (2분):**
   > "이제 전부 경주시켜 봅시다. `examples/a3_sort_benchmark.py`를 실행하세요. 이 스크립트는 여러분이 구현한 모든 정렬 알고리즘을 N=100, 1,000, 10,000, 100,000에서 측정하고 비교 그래프를 생성합니다."

2. **학생 실습 (6분):**
   - `python examples/a3_sort_benchmark.py`를 실행하고 결과를 관찰합니다.
   - N=100에서는 모든 알고리즘이 빠릅니다. N=10,000에서 O(n^2) 알고리즘이 힘들어지기 시작합니다.
   - N=100,000에서 선택 정렬과 버블 정렬은 수십 초가 걸리지만 병합 정렬과 퀵 정렬은 1초 이내에 끝납니다.
   - 학생들에게 질문: "어떤 N부터 차이가 체감되나요?"

3. **체크포인트 (2분):**
   > "교차점은 보통 N=1,000에서 N=10,000 사이입니다. 그 이하에서는 병합 정렬과 퀵 정렬의 재귀 오버헤드 때문에 단순 정렬과 비슷할 수 있습니다. 그 이상에서는 O(n^2)이 고통스러울 정도로 느려집니다. 이것이 실제 애플리케이션에서 올바른 알고리즘을 선택하는 것이 중요한 이유입니다."

## Type B -- 웹 코드 분석

### B-1: 미니 쇼핑몰 정렬 비교 (15분)

#### 진행 방법

1. **간단 소개 (3분):**
   > "온라인 쇼핑몰을 만들고 상품을 가격순으로 정렬해야 한다고 상상해 보세요. `examples/b1_web_sort/`의 Flask 앱을 열어보세요. 두 개의 버튼이 있는 웹 페이지가 있습니다 -- 하나는 버블 정렬로, 다른 하나는 퀵 정렬로 상품을 정렬합니다. 서버를 시작하고 둘 다 해보세요."

   서버 시작 방법을 안내합니다:
   ```
   cd examples/b1_web_sort
   pip install flask
   python app.py
   ```

2. **학생 실습 (9분):**
   - 브라우저에서 `http://localhost:5000`을 엽니다.
   - "버블 정렬로 정렬" 버튼을 클릭하고 로딩 시간을 확인합니다.
   - "퀵 정렬로 정렬" 버튼을 클릭하고 로딩 시간을 확인합니다.
   - 상품 수를 1,000에서 10,000, 50,000으로 변경하며 차이를 체감합니다.
   - 토론 질문: 실제 온라인 쇼핑몰이 O(n^2) 정렬 알고리즘을 사용하면 어떻게 될까요?

3. **체크포인트 (3분):**
   > "1,000개일 때는 두 버튼 모두 빠르게 응답합니다. 50,000개일 때는 버블 정렬이 페이지를 수 초간 멈추게 하지만 퀵 정렬은 즉시 응답합니다. 수백만 개의 상품이 있는 실제 쇼핑몰에서 O(n^2) 정렬을 쓰면 사이트를 사용할 수 없게 됩니다. 이것이 모든 주요 언어와 프레임워크가 내부적으로 O(n log n) 정렬 알고리즘을 사용하는 이유입니다."
